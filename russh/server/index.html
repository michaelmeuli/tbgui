§Writing servers

There are two ways of accepting connections:

  • implement the Server trait and let run_on_socket/run_on_address handle everything
  • accept connections yourself and pass them to run_stream

In both cases, you’ll first need to implement the Handler trait - this is where you’ll handle various events.

Check out the following examples:

  • Server that forwards your input to all connected clients
  • Server handing channel processing off to a library (here, russh-sftp)
  • Serving ratatui based TUI app to clients: per-client, shared
